API dziennika

Cztery proste endpointy REST, które pozwalają deweloperowi lub agentowi AI pisać i czytać wpisy, rozmawiać z asystentem oraz wyszukiwać hybrydowo (słowa + sens) — w imieniu konkretnego użytkownika.

Uwierzytelnianie

        
Każdy endpoint działa per-użytkownik. Klucz API jednoznacznie wskazuje
        Twoje konto — nie trzeba przekazywać żadnego ID użytkownika. Wygeneruj klucz w pasku na
        górze tej strony (po zalogowaniu) i dołączaj go do każdego żądania:

        

          
          
Authorization: Bearer dtm_twoj_klucz

        

        
Klucz jest długożyciowy (nie wygasa, dopóki go nie usuniesz),
        więc agent wkleja go raz. Przechowujemy wyłącznie jego skrót (hash) — pełny klucz widzisz
        tylko w momencie wygenerowania. Brak / zły klucz → 401.

POST /ask

        
Zadaje pytanie asystentowi — empatycznemu towarzyszowi refleksji. Najpierw przeszukuje
        cały dziennik wyszukiwaniem hybrydowym (słowa + znaczenie) pod kątem
        pytania, dobiera najtrafniejsze wpisy (plus kilka najnowszych) i dopiero na ich podstawie
        odpowiada.

        

          
PoleTypOpis

          
questionstringwymaganePytanie do asystenta.

          
match_countinteger 1–50opcjonalneIle najtrafniejszych wpisów wziąć jako kontekst. Domyślnie 12.

          
recent_daysinteger 0–90opcjonalneOkno najnowszych wpisów zawsze doklejanych do kontekstu. Domyślnie 7. 0 = wyłącz.

        

        

          
curl

  "model": "llama-3.1-8b-instant",

POST /search

        
Wyszukiwanie hybrydowe: łączy wyszukiwanie pełnotekstowe (słowa kluczowe)
        z wektorowym (znaczenie/semantyka) i scala wyniki metodą RRF. Dodatkowo do wyników
        zawsze dokleja wpisy z ostatnich recent_days dni (domyślnie 7),
        dla świeżego kontekstu czasowego — nawet jeśli nie pasują do zapytania.

        

          
PoleTypOpis

          
qstringwymaganeZapytanie użytkownika.

          
match_countinteger 1–100opcjonalneIle trafień brać z każdej metody przed scaleniem. Domyślnie 30.

          
recent_daysinteger 0–90opcjonalneOkno kontekstu czasowego doklejanego zawsze. Domyślnie 7. 0 = wyłącz.

        

        

          
curl

          
          

        

        

          
Odpowiedź 200

          
          
{
  "query": "wypalenie w zespole",
  "count": 18,
  "recent_days": 7,
  "results": [
    {
      "id": "2bnganhbximmqak9jbm",
      "title": "",
      "content": "Zaczął się problem wypalenia — ciągła zmiana zabija…",
      "mood": 3,
      "mood_key": "neutral",
      "tags": [],
      "moon_phase": "full",
      "created_at": "2026-05-07T00:00:00+00:00",
      "score": 0.0196,
      "source": "search"
    }
  ]
}

        

        
source mówi, skąd pochodzi wynik:
        search = trafienie wyszukiwania, recent = kontekst z ostatnich dni,
        both = jedno i drugie. Wyniki są posortowane malejąco po score
        (kontekst-only ma score 0 i trafia na koniec).

Serwer MCP

      
Cienka nakładka MCP nad tym samym API — żeby agenci AI (np. Claude)
      korzystali z dziennika przez natywne narzędzia, bez ręcznego sklejania żądań HTTP.


      

        
Czym jest MCP

        
Model Context Protocol to standard, dzięki któremu agent AI
        odkrywa i wywołuje narzędzia serwera. Nasz serwer wystawia trzy narzędzia —
        add_entry, ask_assistant, get_entry — które
        forwardują do tego samego REST API i używają tego samego klucza dtm_…
        (wygeneruj go w pasku na górze). Transport: MCP Streamable HTTP.
